# Coding Requirements

Requirements for all code in this repository.

## General

- All source code, unit tests, and integration tests are generated by Claude based on requirements documents
- Code must be clear, maintainable, and well-structured

## Python

### Style

- Follow PEP 8
- Use type hints for all function signatures
- Maximum line length: 100 characters

### Testing

- Unit tests use pytest
- Test files mirror source structure in `tests/unit/`
- Integration tests in `tests/integration/`
- Minimum coverage target: TBD

### Dependencies

- Minimize external dependencies
- Pin versions in requirements files
- Document why each dependency is needed
